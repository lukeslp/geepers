# Description:
# CLI tools that process files can write derived output (config stubs, credential
# files, summaries) into the test/ directory during manual testing. These runtime
# artifacts can contain sensitive data extracted from test inputs (e.g. API keys
# pulled from an HTML fixture file). They must be gitignored at both the
# package-level and in any root-level .gitignore. Never rely on .gitignore_global
# alone — it is not committed to the repo and won't protect other contributors.
# ================================================

## The Hazard

cleanupx scanned a test HTML file during a manual test run and extracted what
looked like an API key, writing it to:

    test/.xsnippet/api_credentials.txt

Because test/.xsnippet/ was not gitignored at the package level, a naive
`git add -A` from a root repo that contains many packages would have staged
and committed the credential file.

## Fix: .gitignore entries for runtime test output

Add to the **package-level** `.gitignore` (not just global):

```gitignore
# Runtime test output (generated by the tool during manual testing)
test/.xsnippet/
test/cleanupx_output/
test/deduplicated/
test/output/
test/results/
```

## Generalised pattern for any CLI tool

```gitignore
# Any directory the tool writes to during its own operation,
# when that directory might end up inside the test/ tree.
test/<tool-output-dir>/

# Specific known sensitive output patterns
test/**/*credential*
test/**/*api_key*
test/**/*secret*
test/**/*.key
```

## Defence-in-depth checklist

1. Add runtime output dirs to package .gitignore before first test run
2. Add them to root .gitignore if the package lives inside a monorepo
3. Add a conftest.py fixture that creates and cleans up temp dirs:

```python
# conftest.py
import pytest
import shutil
from pathlib import Path

@pytest.fixture(autouse=True)
def clean_test_output(tmp_path):
    """Redirect tool output to pytest's tmp_path, never into test/."""
    yield
    # tmp_path is automatically removed by pytest after the test

@pytest.fixture
def isolated_test_dir(tmp_path):
    """Give each test a clean working directory."""
    return tmp_path
```

4. In CI, verify no credential-looking strings were committed:

```bash
git diff --name-only HEAD~1 HEAD | xargs grep -lE \
  "(api_key|secret|token|credential|password)" 2>/dev/null \
  && echo "WARNING: possible credential in diff" && exit 1 || true
```
