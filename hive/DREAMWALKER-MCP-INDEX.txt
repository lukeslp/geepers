WHAT'S BEING BUILT
================================================================================

A production-ready remote MCP server that exposes the Dreamwalker orchestrator
to Claude Desktop via HTTPS with bearer token authentication.

Key Features:
- 7 MCP Tools exposed (dream_orchestrate_research, dream_orchestrate_search, etc.)
- FastMCP HTTP transport for standard MCP protocol
- Bearer token authentication (non-hardcoded, from environment)
- Health endpoint for monitoring (no auth required)
- Async/await support for long-running workflows
- SSE streaming-ready for real-time progress
- Integration with existing service manager
- Reverse proxy behind Caddy HTTPS

TECHNICAL DECISIONS
================================================================================

1. HYBRID APPROACH (Thin HTTP Wrapper)
   ✅ Create new FastMCP HTTP server

   ✅ Keep unified_server.py unchanged
   ✅ Clean separation of concerns

2. PORT ALLOCATION
   ✅ Port 5059 in 5050-5059 range
   ✅ Verified as available (lsof -i :5059)
   ✅ Localhost only, proxied by Caddy
   ✅ Health endpoint on /health

3. AUTHENTICATION
   ✅ Bearer token in Authorization header
   ✅ Stored in API_KEYS.md (gitignored)
   ✅ Loaded from MCP_BEARER_TOKEN env var
   ✅ Health endpoint exempt (for monitoring)

4. FRAMEWORK
   ✅ FastMCP>=0.1.0 (modern MCP HTTP standard)
   ✅ Uvicorn web server
   ✅ Pydantic for validation
   ✅ Native async/await support

================================================================================
RISK MITIGATION
================================================================================

Risk: Auth bypass
→ Bearer token required, non-hardcoded, from env var

Risk: Timeouts on long tasks
→ Caddy timeout = 10 min, SSE streaming for progress

Risk: Memory leaks
→ Workflow cleanup job, max active workflows limit

Risk: Claude Desktop incompatibility
→ Uses standard FastMCP HTTP spec

Risk: Service availability
→ Health monitoring by service manager

Risk: Accidental data loss
→ Rollback plan documented in deployment-plan.md
