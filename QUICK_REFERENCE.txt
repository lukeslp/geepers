ACCESSIBILITY SKILL DESIGN
═════════════════════════════════════════════════════════════════════════════

Pattern: Orchestrator (like Quality)
Coordinates 4 check types:
  1. WCAG Compliance (semantic HTML, ARIA)
  2. Keyboard Navigation (tab order, focus)
  3. Color Contrast (ratios)
  4. Screen Reader Support (labels, live regions)

Scoring: 0-100 overall score
Output: MD report + HTML dashboard

COMMON PITFALLS
═════════════════════════════════════════════════════════════════════════════

❌ Forgetting to update both platforms (they're separate)
✅ Use single source (manifest), rebuild both

❌ Editing platform files directly (they get regenerated)
✅ Edit manifest.yaml, which is the source

❌ Missing aliases.json entries
✅ Add to both platforms' aliases.json

❌ Incorrect YAML frontmatter
✅ Use provided template, validate with: python3 -c "import yaml; yaml.safe_load(open(...))"

❌ Report output in wrong location
✅ Always use: ~/geepers/reports/by-date/YYYY-MM-DD/
