# Geepers Agent System

`geepers` is a multi-agent orchestration system and development assistant toolset. It is delivered both as a Claude Code plugin and a Python package (`geepers` on PyPI).

## Project Overview

The system consists of 72 specialized agents across 15 domains, designed to assist with coding, debugging, deployment, and research. It leverages the shared infrastructure in `~/shared/` for LLM connectivity.

### Key Components

- **Agent Definitions**: Located in `geepers/agents/`, these Markdown-based files define the persona, tools, and workflows for each agent.
- **Orchestrators**: Located in `geepers/orchestrators/`, these manage how agents interact (e.g., `DreamCascade` for hierarchical tasks).
- **Skills**: Specialized capabilities for Claude Desktop, located in `geepers/skills/`.
- **MCP Integration**: Uses Model Context Protocol to expose tools to LLMs.
- **Recommendations**: Project-specific guidance generated by agents, found in `geepers/recommendations/`.

## Building and Running

### Plugin Installation
To add geepers as a plugin to Claude Code:
```bash
/plugin add lukeslp/geepers
```

### Python Usage
To use geepers in Python scripts:
```python
from geepers import DreamCascadeOrchestrator
orchestrator = DreamCascadeOrchestrator()
```

### Development
- **Testing**: `pytest geepers/tests`
- **Agent Registry**: Managed via `geepers/agents/registry.json`.

## Development Conventions

- **Persona Consistency**: All agents should follow the established tone and technical standards defined in the base agent templates.
- **Tool Usage**: Agents should prefer using defined MCP tools for interacting with the environment.
- **Domain Focus**: New agents should be assigned to one of the 15 established domains (Search, Web, Frontend, etc.).

## Key Files
- `agents/`: Markdown persona definitions.
- `geepers/`: Core Python logic.
- `skills/`: Claude Desktop skill definitions.
- `QUICK_REFERENCE.md`: Summary of all agents and their tasks.
